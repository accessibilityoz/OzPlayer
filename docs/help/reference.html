Player Container

The player container is a wrapper element that holds all the others together, while defining a reference to the transcript container, and specifying how the controls are laid out:

  1. "id" (required)

    The player container ID, which can be anything you like.

    The ID is used by JavaScript to initialise the player.

  2. "class" (required)

    The player container class, which must have the exact value "ozplayer".

    The class is used as a descendent root for much of the player's CSS.

  3. "data-transcript" (recommended)

    The transcript ID reference, which specifies the ID of the transcript container, if you have one.

    If you don't have a transcript then you should omit this attribute. However the use of a transcript is strongly encouraged, for best accessibility and user engagement.

  4. "data-responsive" (optional)

    The responsive container ID reference, which can specify a reference to any page element, that OzPlayer will adapt itself to.

    If this attribute is specified, and points to a flexible container, then OzPlayer will continually resize itself to match the internal width of the container (down to an absolute minimum of 240 pixels).

  5. "data-responsive-mode" (optional)

    The responsive sizing mode, which can be "min", "max" or "initial".

    If it has the value "min" then the default video width will also be the minimum width, when resizing to fit a responsive container.

    If it has the value "max" then the default video width will also be the maximum width.

    If it has the value "initial" then there won't be any minimum or maximum width constraints (apart from the absolute minimum of 240 pixels).

    If this attribute is not defined, the mode defaults to "initial".

  6. "data-controls" (optional)

    The player controls layout, which can either be "stack" or "row".

    If it has the value "stack" then the controls will be superimposed over the bottom of the video, with a timer that automatically hides them after a few seconds of user inactivity.

    If it has the value "row" then the controls will be a separate row, which is positioned underneath the video and remains permanently visible.

    If this attribute is not defined, the layout defaults to "row".

Video Element

The video element defines the video's default appearance and layout, and controls how preloading should be handled for both the video and audio descriptions:

  1. "width" (recommended)

    The video width in pixels, which must be 240 or more.

    If this attribute is not defined, the width defaults to 400.

  2. "height" (recommended)

    The video height in pixels, which must be 135 or more.

    If this attribute is not defined, the height defaults to 225.

  3. "poster" (optional)

    The video poster image, which specifies the path to an image file, usually a JPEG or PNG.

    If a poster is defined, it will be shown as a video placeholder until playback starts, with a large click to play icon in the middle. The poster will be automatically scaled to match the video's width and height

    If a poster is not defined, then the video will show a black background (except for YouTube videos).

  4. "preload" (recommended)

    The video and audio preload setting, which can be "auto" or "none".

    If preload is "auto" then the video and audio will start preloading as soon as the page loads.

    If preload is "none" then no preloading will occur until the user first presses Play. Once playback has started, it will continue to load in the background either way.

    If this attribute is not defined, preload defaults to "none".

  5. "controls" (required)

    This is a boolean attribute, which tells the browser to provide native controls for the video.

    Although native controls are removed once the scripted controls are added, this attribute is still important to ensure that the video can be played without scripting.

Video Source Elements

One or more video sources define the path and mime-type of each of the files that the player will be using:

  1. "src" (required)

    The path to a video file.

  2. "type" (required)

    The video file's mime-type, such as "video/mp4".

YouTube Source Element

For YouTube videos, a single source defines the URL and a proprietary type, and optionally controls the appearance of embedded YouTube captions:

  1. "src" (required)

    Specifies a YouTube URL, such as "http://www.youtube.com/watch?v=12345".

    It can also use a short-link such as "http://youtu.be/12345" or an embeddable URL like "//www.youtube.com/embed/12345".

  2. "type" (required)

    This must have the exact value "video/x-youtube", which is a proprietary value used by the script to identify YouTube videos.

  3. "data-captions" (optional)

    Adding this attribute specifies that embedded YouTube captions should be controllable via the player interface.

    If present, the value of this attribute controls whether the embedded captions should be enabled ("true") or disabled ("false") by default.

Captions Track Elements

One or more caption-track elements define the data to use for the captions and transcript, and specify whether the captions should be shown by default:

You can specify multiple languages by adding multiple tracks:

  1. "src" (required)

    The path to a captions cues file, in WebVTT or SRT format.

  2. "kind" (required)

    Specifies what kind of track this is, which must have the exact value "captions".

  3. "data-sync" (optional)

    The synchronisation offset, which can specify a timing adjustment in seconds.

    If this attribute is defined, as an integer or floating-point number, then its value will be added to the timing of every cue. For example, if it has the value "3.5", then a cue defined for "02.000 --> 03.000" would appear at "05.500 --> 06.500".

  4. "srclang" (required)

    The ISO language code of the captions data.

  5. "label" (required for multiple tracks)

    A short label for the captions track.

    If multiple tracks are included in different languages, then the labels will be used to create a menu for users to choose between them.

  6. "default" (recommended)

    This is a boolean attribute, which specifies whether the captions should be shown by default.

    If this attribute is omitted, then the captions will be off by default.

  7. "data-default-transcript" (recommended for multiple tracks if no default)

    This is a boolean attribute, which specifies which track should be used to create the default transcript output, in cases where multiple languages are defined, but captions are off by default.

    If none of the tracks have this attribute then the first one will be used.

Transcript Track Elements

One or more transcript-track elements can define additional cues that only appear in the transcript. These will be merged with the captions cues according to their timings:

  1. "src" (required)

    The path to a transcript cues file, in WebVTT or SRT format.

  2. "kind" (required)

    Specifies what kind of track this is, which must have the exact value "metadata".

  3. "data-kind" (required)

    Specifies what kind of meta-data this is, which must have the exact value "transcript".

  4. "data-sync" (optional)

    The synchronisation offset, which can specify a timing adjustment in seconds.

    This works in exactly the same way as the captions track data-sync attribute.

  5. "srclang" (required)

    The ISO language code of the transcript data.

    When multiple tracks are included in different languages, the data from each transcript track will be merged with the captions data that matches its language code.

Fallback Content Container

The fallback content container is a wrapper for any content you want to show to older browsers, i.e. those which don't support the video sources or the Flash replacement:

  1. "class" (required)

    The fallback container class, which must have the exact value "ozplayer-fallback".

    The class can used as a descendent root for any CSS you want to define for the fallback content.

Audio-Only Element

The audio element defines the audio-only player's default appearance and layout, and controls how preloading should be handled:

  1. "data-width" (recommended)

    The player width in pixels, which must be 240 or more.

    If this attribute is not defined, the width defaults to 400.

  2. "preload" (recommended)

    The audio preload setting, which can be "auto" or "none".

    If preload is "auto" then the audio will start preloading as soon as the page loads.

    If preload is "none" then no preloading will occur until the user first presses Play. Once playback has started, it will continue to load in the background either way.

    If this attribute is not defined, preload defaults to "none".

  3. "controls" (required)

    This is a boolean attribute, which tells the browser to provide native controls for the audio.

    Although native controls are removed once the scripted controls are added, this attribute is still important to ensure that the audio can be played without scripting.

Audio-Only Source Elements

One or more audio sources define the path and mime-type of each of the files that the player will be using:

  1. "src" (required)

    The path to an audio file.

  2. "type" (required)

    The audio file's mime-type, such as "audio/mp3".

Audio-Only Captions Track Elements

These are defined in exactly the same way as video captions track elements.

Audio-Only Transcript Track Elements

These are defined in exactly the same way as video transcript track elements.

Audio-Only Fallback Content Container

This is defined in exactly the same way as the video fallback content container.

Audio Descriptions Element

The audio element specifies whether audio descriptions are enabled by default, and controls preloading for browsers which don't support them:

  1. "data-default" (recommended)

    This is a boolean attribute, which specifies whether the audio descriptions should be enabled by default.

    If this attribute is omitted, then the audio descriptions will be off by default.

  2. "preload" (recommended)

    The audio preload setting, which should be set to "none" to prevent preloading in browsers which don't support audio descriptions.

    For browsers that do support audio descriptions, the video's preload setting will be used, so the audio preload is only used by browsers which don't support them, or where scripting is unavailable.

Audio Descriptions Source Elements

One or more audio sources define the path and mime-type of each of the files that the player will be using:

  1. "src" (required)

    The path to an audio file.

  2. "type" (required)

    The audio file's mime-type, such as "audio/mp3".

Audio Linking Element

The audio element can alternatively be used as a means of linking between two pages addresses, one of which has a version of the video with audio descriptions built-in:

  1. "data-off" (required)

    The address of a page containing the non-described version of the video.

  2. "data-on" (required)

    The address of a page containing the audio-described version of the video.

Transcript Container

The transcript container is an empty element, which the generated transcript will be built inside:

  1. "id" (required)

    The transcript container ID, which can be anything you like, but must be the same as the data-transcript attribute defined for the player container.

  2. "class" (required)

    The transcript container class, which must have the exact value "ozplayer-transcript".

    The class is used as a descendent root for all the transcript CSS defined in transcript.css.

Transcript Expander

The transcript expander is an optional parent element of the transcript container, which makes it possible to expand and collapse the transcript:

  1. "class" (recommended)

    The expander class, which must have the exact value "ozplayer-expander".

    If the class is omitted, then the native behaviour of the DETAILS element will not be polyfilled for older browsers.

  2. "open" (recommended)

    This is a boolean attribute, which specifies whether the transcript should be expanded by default.

    If this attribute is omitted, then the transcript will be collapsed by default.
