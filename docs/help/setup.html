OzPlayer 4.0 Setup Guide

OzPlayer uses HTML5 to define the video, audio, captions and transcript data it uses. This guide introduces the elements you need to create a working player, with the best possible accessibility and graceful degradation.

Adding Richer Semantics

The containers should be DIV elements, since they're just generic containers.

But you can add richer semantic by wrapping them both in a FIGURE element. This can optionally have a FIGCAPTION element — usually with the video's title — while the FIGURE itself can be used as a hook for additional layout styles (e.g. for centering the player):

If you want to expand and collapse the transcript you can wrap it in a DETAILS element, and OzPlayer will automatically polyfill its behaviour for older browsers:

The DETAILS element must have a first-child SUMMARY with visible text, which is then immediately followed by the transcript container. If the DETAILS has an open attribute then the transcript will be expanded by default, otherwise it will be collapsed. If you omit the "ozplayer-expander" class then the native behaviour will not be polyfilled.

Using Responsive Design

OzPlayer can adapt to responsive design — if it's placed within a flexible column that the player can identify. For example, if the player is inside the main "content" column, then you would specify that column's ID using the player's data-responsive attribute:

OzPlayer will then resize itself to fit that content column. This is measured during page load, and whenever the window size or orientation changes.

Adding the Video

The only required attribute is controls, which tells the browser to provide native controls when JavaScript is unavailable. This is important to ensure that the video can still be played without scripting (if the scripting fails to load, or is blocked or disabled).

Adding Captions

Captions can be added using a TRACK element, which goes inside the VIDEO or AUDIO element. The captions will appear at the bottom of a video, and are also used to generate the transcript:

Each track has three required attributes — an src which points to a captions file, a kind attribute which must have the value "captions", and an srclang which specifies the ISO language code. The default attribute specifies that the captions should be switched on by default, however this is optional.

You can also specify multiple languages by adding multiple tracks, each with different src, srclang and label attributes, and the default attribute used to specify which language is shown by default:

Adding Extra Transcript Data

You can add another TRACK to define additional transcript cues — which will be added to the transcript, but won't be shown as captions:

Each track has required src and srclang attributes, the same as the captions track, but the kind attribute in this case must have the value "metadata", and there's also a data-kind attribute which must have the value "transcript".
